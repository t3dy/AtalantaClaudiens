The deploy workflow (`deploy.yml`) only uploads `site/`. It does NOT run the Python pipeline. This is correct for the current architecture (local build, commit generated files), but means:
- Database rebuild must happen locally before push
- No automated validation in CI
- If someone pushes without rebuilding, stale HTML goes live
